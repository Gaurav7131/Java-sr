1. Classes and Objects: The Blueprint and the Toy
Remember that a Class is just a blueprint for a brand-new robot toy, not the real robot itself. The Object is the actual toy you build, like a red robot named "Sparky".

The Code Snippet:

Java
// The Class (The Blueprint)
class Robot {
    // Variables (State)
    String name;
    String color;

    // Method (Behavior)
    void walk() {
        System.out.println(name + " is walking!");
    }
}

public class Main {
    public static void main(String[] args) {
        // The Object (The Real Toy)
        Robot sparky = new Robot(); 
        sparky.name = "Sparky";
        sparky.color = "Red";
        
        sparky.walk(); // Output: Sparky is walking!
    }
}
Common Interview Question: > Q: What is the difference between a Class and an Object?


A: A Class is a blueprint or template that defines the state and behavior of an entity. An Object is an actual instance of that class that takes up memory and has specific values assigned to its state.

2. Variables and Data Types: The Storage Boxes
Variables are like cardboard boxes used to remember things. Java is very strict about what goes in the box; you can't put a wet swimsuit in a box meant for dry books.

The Code Snippet:

Java
public class Main {
    public static void main(String[] args) {
        // Strict data types for specific boxes
        int age = 10;                  // strictly for whole numbers [cite: 18]
        String message = "Hello World"; // strictly for words and text [cite: 19]
        boolean isLightOn = true;       // tiny box that only holds true or false [cite: 20]
    }
}
Common Interview Question:

Q: Java is a "statically-typed" language. What does that mean?


A: It means every variable must be declared with a specific data type before it can be used, and it can only hold values of that exact type. The compiler checks this before the program even runs.

3. Methods: The Action Buttons
If variables are the things you have, methods are the things you do. Instead of rewriting jumping instructions every time, you just call the jump() method like pressing a button on a controller.

The Code Snippet:

Java
class Player {
    // The Method (Action Button)
    void jump() {
        System.out.println("Character jumps into the air!");
    }
}

public class Game {
    public static void main(String[] args) {
        Player mario = new Player();
        mario.jump(); // "Pressing" the button to trigger the action
    }
}
Common Interview Question:

Q: Why do we use methods in Java?
A: Methods promote code reusability. Instead of writing the same logic multiple times, you write it once in a method and simply call that method whenever the action is needed.

4. Conditionals (If / Else): The Crossroads
Computers use "If/Else" statements to make decisions based on rules. IF it is raining, take an umbrella; ELSE, wear a sun hat.

The Code Snippet:

Java
public class Weather {
    public static void main(String[] args) {
        boolean isRaining = true;

        if (isRaining) {
            System.out.println("Take an umbrella.");
        } else {
            System.out.println("Wear a sun hat.");
        }
    }
}
Common Interview Question:

Q: When would you use an if-else statement versus a switch statement?
A: An if-else statement evaluates boolean conditions (true/false) and is great for complex logic or ranges. A switch statement evaluates a single variable against multiple specific, fixed values (like matching a specific day of the week), making the code cleaner to read.

5. Loops (For / While): The Repeat Machine
Loops act as a magical machine that repeats a chore over and over, like writing "I will not talk in class" 100 times on a chalkboard.

The Code Snippet:

Java
public class Punishment {
    public static void main(String[] args) {
        // A 'for' loop that runs exactly 100 times
        for (int i = 1; i <= 100; i++) {
            System.out.println(i + ": I will not talk in class");
        }
    }
}
Common Interview Question:

Q: What is the difference between a for loop and a while loop?
A: A for loop is typically used when you know exactly how many times you want to iterate (like exactly 100 times). A while loop is used when you want the code to repeat until a specific condition changes, regardless of how many times it takes.

6. The JVM: The Universal Translator
The JVM (Java Virtual Machine) is the magical translator that gives Java its "Write Once, Run Anywhere" superpower.

Common Interview Question:

Q: How does Java achieve platform independence?
A: When you write Java code, it gets compiled into "bytecode". This bytecode isn't tied to Windows, Mac, or Linux. Instead, the JVM on the user's specific machine reads that bytecode and translates it into machine code on the spot. Because every operating system has its own JVM, you only have to write your Java code once!

The Scenario: Imagine you have a piggy bank. You don’t want your little brother to just reach in and grab your money! So, the piggy bank is sealed tight. The only way to put money in or check your balance is through the tiny coin slot on top.

In Java, Encapsulation means hiding the important data inside a Class so other parts of the program can't accidentally break it or steal it. You make the data private, and you give people safe public methods (like the coin slot) to interact with it.

class Piggy {

     * public method 1)addcoin only addcoin and checkbal permits rather than balance

     */

        }

    // 2)checkbalace:check money
    public int checkbalance() {
        return balance;
    }
}

public class Encapsulation {
    public static void main(String[] args) {

3. Polymorphism: The Universal Video Game Button
The Scenario:
"Poly" means many, and "morph" means forms. Think of a video game controller. You have an "Attack" button. If you are playing as a Knight, pressing Attack makes him swing a sword. If you are playing as a Wizard, pressing that exact same button makes him shoot a fireball.

In Java, Polymorphism means you can call the same method, but it will do different things depending on which specific Object is performing the action.


class VideoGameChar {
    void attack() {
        System.out.println("Attack button basics features");
    }
}

class Knight extends VideoGameChar {
    @Override
    void attack() {
        System.out.println("Knight swing the sword");
    }
}

class Wizard extends VideoGameChar {
    @Override
    void attack() {
        System.out.println("Wizards throwing fireball");
    }
}

public class Polymorphism {
    public static void main(String[] args) {
        Knight kn = new Knight();
        kn.attack();

        Wizard wz = new Wizard();
        wz.attack();

    }
}

1. Q: Is Java Pass by Value or Pass by Reference? (सर्वात जास्त विचारला जाणारा प्रश्न)

A: Java is strictly Pass by Value. It does not support Pass by Reference. When we pass a primitive data type, a copy of the value is passed. When we pass an object, a copy of the reference (memory address) is passed, not the actual object itself.

2. Q: If Java is Pass by Value, why does changing an object's property inside a method change the original object?

A: Because when you pass an object to a method, you are passing a copy of its reference. Both the original reference and the copied reference point to the exact same object in the memory (Heap). So, modifying the object's state using the copied reference will affect the original object.

3. Q: Can you swap two objects in a method in Java?

A: No. Since Java passes a copy of the reference, if you try to swap the references inside a method (e.g., making reference A point to object B), it only changes the copied references. The original references back in the main method remain completely unchanged.

सोपे उदाहरण: समजा तुझ्याकडे एक मोठी रशियन बाहुली आहे. तू ती उघडतोस, तर आतमध्ये तशीच पण थोडी छोटी बाहुली मिळते. तू ती छोटी बाहुली उघडतोस, तर आत आणखी छोटी बाहुली मिळते! हे असंच चालू राहतं जोपर्यंत सर्वात छोटी बाहुली (जी उघडता येत नाही) सापडत नाही.

Java मध्ये, जेव्हा एखादी Method (अॅक्शन बटन) स्वतःलाच पुन्हा पुन्हा हाक मारते (Call करते), तेव्हा त्याला 'Recursive Method' म्हणतात. म्हणजेच, एक मोठं काम पूर्ण करण्यासाठी ती मेथड स्वतःलाच एका छोट्या कामासाठी बोलावते.

२. थांबण्याची अट (The Base Condition) - सर्वात महत्त्वाची गोष्ट!
जेव्हा आपण आरशासमोर दुसरा आरसा ठेवतो, तेव्हा प्रतिमा अनंत (Infinite) काळ दिसत राहते. कोडिंगमध्ये असं झालं तर कॉम्प्युटर हँग होईल! (ज्याला प्रोग्रामिंगच्या भाषेत StackOverflowError म्हणतात).

म्हणून प्रत्येक Recursive Method मध्ये 'थांबण्याची अट' (Base Condition) द्यावीच लागते. (जसे की - सर्वात छोटी बाहुली सापडली की थांबणे).
>>Recursive method>>BAse Cond>return>once callled recmethod

public class Rocket {

    // ही आपली Recursive Method आहे
    static void countdown(int number) {
        
        // १. थांबण्याची अट (Base Condition) - जर नंबर ० झाला, तर थांब!
        if (number == 0) {
            System.out.println("Blast Off! रॉकेट उडाले! 🚀");
            return; // इथून मेथड पूर्णपणे थांबते आणि बाहेर येते
        }

        // २. सध्याचा नंबर प्रिंट कर
        System.out.println(number);

        // ३. Recursion (जादू!) - मेथड स्वतःलाच पुन्हा हाक मारत आहे, पण एका कमी नंबरने!
        countdown(number - 1);
    }

    public static void main(String[] args) {
        System.out.println("रॉकेट उड्डाणाची तयारी सुरू:");
        countdown(5); // ५ पासून सुरुवात करा
    }
}

🎯 Recursion: Interview Questions (मुलाखतीचे प्रश्न)
1. Q: What is Recursion in Java?

A: Recursion is a process in which a method calls itself continuously to solve a smaller piece of the same problem.

मराठीत अर्थ: जेव्हा एखादी Method स्वतःलाच पुन्हा पुन्हा हाक मारते, तेव्हा त्याला Recursion म्हणतात.

2. Q: What is a "Base Condition" (or Base Case) in recursion, and why is it important?

A: A base condition is the stopping point for the recursive method. It is important because without it, the method will keep calling itself infinitely.

मराठीत अर्थ: ही एक 'थांबण्याची अट' आहे. ही अट दिली नाही तर मेथड स्वतःला हाक मारणे कधीच थांबवणार नाही.

3. Q: What happens if you don't provide a Base Condition in a recursive method?

A: The program will result in infinite recursion. It will quickly run out of memory and crash by throwing a StackOverflowError.

मराठीत अर्थ: प्रोग्राम अनंत काळ चालत राहील आणि कॉम्प्युटरची मेमरी भरून तो StackOverflowError देईल.

4. Q: Recursion vs Iteration (Loops) - Which one is better?

A: Iteration (using loops like for or while) is generally faster and uses less memory. However, Recursion is preferred when solving complex mathematical problems or working with complex data structures (like Trees and Graphs) because it makes the code much cleaner and easier to read.

मराठीत अर्थ: Loops मेमरी वाचवतात आणि फास्ट असतात. पण जेव्हा खूप गुंतागुंतीचे प्रॉब्लेम्स सोडवायचे असतात, तेव्हा Recursion मुळे कोड खूप सोपा आणि वाचायला सुटसुटीत होतो.

5. Q: Can you give a real-world example of recursion?

A: Yes! A perfect example is a set of Russian Nesting Dolls, where opening one doll reveals a smaller, identical doll inside, until you reach the smallest one that cannot be opened (which acts as the base condition).

Java>>DIdinot excute
static int findMax(int[] arr) {
    int max = arr[0]; // पहिल्या मुलाला राजा मानले
    for (int num : arr) {
        if (num > max) {
            max = num; // जर कुणी मोठा सापडला, तर त्याला नवीन राजा बनवले!
        }
    }
    return max;
}
B. Extend Array (अॅरे मोठा करणे)

सोपे उदाहरण: तुझा ५ चॉकलेट्सचा डबा भरला आहे. तू बाजारातून १० चॉकलेट्स मावणारा नवीन मोठा डबा आणतोस आणि जुने चॉकलेट्स त्यात टाकतोस.

Java
static int[] extendArray(int[] oldArray, int newSize) {
    int[] newArray = new int[newSize]; // नवीन मोठा डबा
    for (int i = 0; i < oldArray.length; i++) {
        newArray[i] = oldArray[i]; // जुने सामान नवीन डब्यात टाकले
    }
    return newArray;
}

static String capitalizeWords(String str) {
    String[] words = str.split(" ");
    StringBuilder result = new StringBuilder();
    for (String word : words) {
        // पहिले अक्षर मोठे (Capital) करणे आणि उरलेला शब्द जोडणे
        result.append(Character.toUpperCase(word.charAt(0)))
              .append(word.substring(1)).append(" ");
    }
    return result.toString().trim();
}

static void drawEmptyRectangle(int width, int height) {
    for (int i = 1; i <= height; i++) {
        for (int j = 1; j <= width; j++) {
            // जर पहिली/शेवटची लाईन असेल किंवा पहिली/शेवटची भिंत असेल तर * काढा
            if (i == 1 || i == height || j == 1 || j == width) {

                System.out.print("  "); // नाहीतर रिकामी जागा सोडा
            }
        }

    }

B. Convert Decimal into Roman Numbers (नंबरला रोमन अंकात बदलणे)

सोपे उदाहरण: दुकानातून सुट्टे पैसे घेणे. जर तुला १४ रुपये द्यायचे असतील, तर तू आधी १० ची नोट देशील (X) आणि मग ४ रुपये (IV) देशील.

static String intToRoman(int num) {
    // आपल्याकडील नोटा आणि त्यांची रोमन नावे
    int[] values = {1000, 900, 500, 400, 100, 90, 50, 40, 10, 9, 5, 4, 1};
    String[] romanLetters = {"M","CM","D","CD","C","XC","L","XL","X","IX","V","IV","I"};
    
    StringBuilder roman = new StringBuilder();
    
    for (int i = 0; i < values.length; i++) {
        while (num >= values[i]) {
            num -= values[i]; // पैसे वजा करा
            roman.append(romanLetters[i]); // रोमन नाव जोडा
        }
    }
    return roman.toString();
}


१. Enumeration (Enum) म्हणजे नक्की काय?
सोपे उदाहरण: समजा तू एका दुकानात टी-शर्ट घ्यायला गेलास. तिथे फक्त ३ साईज मिळतात - SMALL, MEDIUM, आणि LARGE. तू दुकानदाराला "मला X-SMALL किंवा सुपर-लार्ज द्या" असं म्हणू शकत नाहीस. जे ठरलेलं आहे, तेच मिळणार!

Java मध्ये जेव्हा तुला एखादी 'फिक्सड लिस्ट' (Fixed List) बनवायची असते, जिच्या बाहेरचे पर्याय कोणीही निवडू नयेत, तेव्हा आपण Enum वापरतो.

Enum Type: हा एक खास प्रकारचा Data Type आहे, जो फक्त तू ठरवून दिलेलेच पर्याय (Constants) साठवतो.

Enum: Interview Questions (मुलाखतीचे प्रश्न)
मुलाखतीत Enum वर हमखास प्रश्न विचारले जातात. खालील इंग्रजी प्रश्न आणि त्यांची सोपी उत्तरे डोक्यात फिट करून ठेव:

1. Q: What is an Enum in Java and why do we use it?

A: Enum is a special data type used to define a collection of constants (fixed values) like days of the week or directions. We use it to ensure type safety so that no invalid values can be assigned.

मराठीत अर्थ: फिक्सड व्हॅल्यूज (उदा. आठवड्याचे वार) साठवण्यासाठी Enum वापरतात, जेणेकरून कोणीही चुकीची माहिती टाकू नये.

2. Q: Can you create an object of an Enum using the new keyword?

A: No, we cannot use the new keyword with Enum. Enum constants are created automatically by the JVM.

मराठीत अर्थ: नाही! आपण Enum चे नवीन Object बनवू शकत नाही. JVM स्वतःहून ते बनवून देतो.

3. Q: How do you compare two Enum constants? Can we use ==?

A: Yes, using the == operator is the recommended way to compare Enum constants because Enums are constants and there is only one instance of each Enum value in memory.

4. Q: What is the use of the .values() method in Enum?

A: The .values() method returns an array containing all the constants of the Enum in the order they were declared. It is mostly used to iterate (loop) over the Enum.

5. Q: Can an Enum implement an interface or extend a class?

A: An Enum can implement interfaces, but it cannot extend any other class because every Enum in Java internally extends the java.lang.Enum class (and Java does not support multiple class inheritance).
